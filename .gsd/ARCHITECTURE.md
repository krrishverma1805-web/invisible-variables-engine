## Overview

The Invisible Variables Engine (IVE) is a production-grade data science system that discovers hidden latent variables in datasets by analyzing systematic model prediction errors. It ingests tabular data, profiles it, trains baseline models, analyzes residuals, and synthesizes new explanatory features.

```
┌─────────────────────────────────────────┐
│     Streamlit UI & REST API (FastAPI)   │
├─────────────────────────────────────────┤
│         Celery Workers (ML Pipeline)    │
│    Data Ingestion -> Profiling -> Core  │
├─────────────────────────────────────────┤
│       PostgreSQL DB & Redis Cache       │
│           Local/S3 Artifact Store       │
└─────────────────────────────────────────┘
```

## Components

### API Layer

- **Purpose:** Exposes REST endpoints for dataset upload, experiment management, and latent variable retrieval. Also provides WebSockets for real-time progress.
- **Location:** `src/ive/api/`
- **Dependencies:** FastAPI, Pydantic, SQLAlchemy

### Data Layer (Ingestion & Profiling)

- **Purpose:** Parses CSVs (using pandas/polars), auto-detects column types, validates data, and generates comprehensive statistical profiles and quality scores.
- **Location:** `src/ive/data/`
- **Dependencies:** pandas, polars, numpy

### Worker Layer

- **Purpose:** Executes heavy ML pipelines asynchronously.
- **Location:** `src/ive/workers/`
- **Dependencies:** Celery, Redis, psycopg2

### Core ML Engine

- **Purpose:** Trains baseline models, computes residuals, performs clustering (HDBSCAN), and synthesizes new latent variables.
- **Location:** `src/ive/core/`, `src/ive/detection/`, `src/ive/construction/`
- **Dependencies:** scikit-learn, xgboost, shap, hdbscan, statsmodels

### Storage Layer

- **Purpose:** Abstracted storage for artifacts (datasets, models) supporting local and S3 backends. Relational data stored in PostgreSQL.
- **Location:** `src/ive/storage/`, `src/ive/db/`
- **Dependencies:** aiofiles, boto3, SQLAlchemy, asyncpg

## Data Flow

1. User uploads CSV via Streamlit UI to FastAPI `/datasets` endpoint.
2. `DataIngestionService` parses, validates, and stores the file in `ArtifactStore`, creating a `Dataset` DB record.
3. User initiates an experiment; FastAPI creates an `Experiment` record and queues a Celery task.
4. Celery worker executes the ML pipeline (Understand -> Model -> Detect -> Construct), updating progress via DB.
5. Streamlit UI polls progress via WebSocket and fetches final latent variables from FastAPI.
